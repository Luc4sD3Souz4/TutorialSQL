O que é?

    A linguagem SQL (Structured Query Language) é o recurso mais conhecido por DBAs e programadores para a execução de comandos em bancos de dados relacionais, permite acessar e manipular bancos de dados. tornando-se um padrão do American National Standards Institute (ANSI) em 1986, e da International Organization for Standardization (ISO) em 1987. O SQL é o recurso mais conhecido por DBAs e programadores para a execução de comandos em bancos de dados relacionais. É por meio dela que criamos tabelas, colunas, índices, atribuímos permissões a usuários, bem como realizamos consultas a dados. Enfim, é utilizando a SQL que “conversamos” com o banco de dados. Essa breve descrição já deixa clara a importância da compreensão e domínio sobre tal recurso. Saiba, também, que ela facilita não apenas a execução de tarefas em SGBDs, mas também o diálogo entre profissionais de banco de dados e programadores. Com base nisso, aprender sobre SQL, a Structured Query Language, passa a ser fundamental para qualquer um que deseja atuar nessas áreas.



    Organização da SQL:



  • DQL - Linguagem de Consulta de Dados - Define o comando utilizado para que possamos consultar (SELECT) os dados armazenados no banco;


  • DML - Linguagem de Manipulação de Dados - Define os comandos utilizados para manipulação de dados no banco (INSERT, UPDATE e DELETE);


  • DDL - Linguagem de Definição de Dados - Define os comandos utilizados para criação (CREATE) de tabelas, views, índices, atualização dessas estruturas (ALTER), assim como a remoção (DROP);


  • DCL - Linguagem de Controle de Dados - Define os comandos utilizados para controlar o acesso aos dados do banco, adicionando (GRANT) e removendo (REVOKE) permissões de acesso;


  • DTL - Linguagem de Transação de Dados - Define os comandos utilizados para gerenciar as transações executadas no banco de dados, como iniciar (BEGIN) uma transação, confirmá-la (COMMIT) ou desfazê-la (ROLLBACK).

    • Pra que serve?

    • SQL pode executar consultas em um banco de dados
    • SQL pode recuperar dados de um banco de dados
    • SQL pode inserir registros em um banco de dados
    • SQL pode atualizar registros em um banco de dados
    • SQL pode excluir registros de um banco de dados
    • SQL pode criar novos bancos de dados
    • SQL pode criar novas tabelas em um banco de dados
    • SQL pode criar procedimentos armazenados em um banco de dados
    • SQL pode criar visualizações em um banco de dados
    • SQL pode definir permissões em tabelas, procedimentos e visualizações

    SQL Select

    O comando select é o “comando base” para todos os comandos de pesquisa de informações no banco de dados.

    SQL Where

    O comando de restrição “where” é o comando de restrição mais utilizado. O comando “where” quer dizer “onde”, ou seja, com a utilização do comando “where”, serão exibidas as informações solicitadas, onde as mesmas obedecerem as restrições informadas.

    SQL Distinct

    O comando “distinct” consiste em exibir as informações da tabela utilizada de maneira distinta, de forma que informações não sejam exibidas de maneira redundante.

    SQL From

    O comando “from” significa “de”, ou seja, indica de onde algo está sendo chamado.

    SQL Insert Into

    A instrução INSERT INTO é usada para inserir novos registros em uma tabela.

    SQL Update

    A instrução UPDATE é usada para modificar os registros existentes em uma tabela.

    SQL Delete

    A instrução DELETE é usada para excluir registros existentes em uma tabela.

    SQL Select Top

    A cláusula SELECT TOP é usada para especificar o número de registros a serem retornados. A cláusula SELECT TOP é útil em tabelas grandes com milhares de registros. Retornar um grande número de registros pode afetar o desempenho.

    SQL Min

    A função MIN () retorna o menor valor da coluna selecionada.

    SQL Max

    A função MAX () retorna o maior valor da coluna selecionada.

    SQL Count

    A função COUNT () retorna o número de linhas que corresponde a um critério especificado.

    SQL Avg

    A função AVG () retorna o valor médio de uma coluna numérica.

    SQL Sum

    A função SUM () retorna a soma total de uma coluna numérica.

    SQL Like

    O operador LIKE é usado em uma cláusula WHERE para procurar um padrão especificado em uma coluna. Existem dois curingas usados ​​em conjunto com o operador LIKE: % - O sinal de porcentagem representa zero, um ou vários caracteres _ - O sublinhado representa um único caractere

    SQL In

    O operador IN permite que você especifique vários valores em uma cláusula WHERE. O operador IN é uma abreviação de várias condições OR.

    SQL Between

    O operador BETWEEN seleciona valores dentro de um determinado intervalo. Os valores podem ser números, texto ou datas. O operador BETWEEN é inclusivo: os valores inicial e final são incluídos.

    SQL Aliases

    Os aliases SQL são usados para fornecer uma tabela ou uma coluna em uma tabela, um nome temporário. Os aliases costumam ser usados para tornar os nomes das colunas mais legíveis. Um alias existe apenas para a duração da consulta.

    SQL Having

    A cláusula HAVING foi incluída no SQL porque a palavra-chave WHERE não pôde ser usada com funções agregadas.

    SQL Exists

    O operador EXISTS é usado para testar a existência de qualquer registro em uma subconsulta. O operador EXISTS retornará true se a subconsulta retornar um ou mais registros.

    SQL Any

    Os operadores ANY e ALL são usados ​​com uma cláusula WHERE ou HAVING. O operador ANY retornará true se algum dos valores da subconsulta atender à condição.

    SQL All

    O operador ALL retorna true se todos os valores da subconsulta atenderem à condição.

    SQL Case

    A instrução CASE passa por condições e retorna um valor quando a primeira condição é atendida (como uma instrução IF-THEN-ELSE). Então, uma vez que uma condição é verdadeira, ela irá parar de ler e retornar o resultado. Se nenhuma condição for verdadeira, retornará o valor na cláusula ELSE. Se não houver nenhuma parte do ELSE e nenhuma condição for verdadeira, ela retornará NULL.

    SQL Order By

    O comando “order by” ordena todas as informações de uma tabela em ordem crescente;

    SQL Order By Desc

    O comando “order by desc” ordena todas as informações de uma tabela em ordem decrescente.

    SQL Group By

    A instrução GROUP BY é frequentemente usada com funções agregadas (COUNT, MAX, MIN, SUM, AVG) para agrupar o conjunto de resultados em uma ou mais colunas.

    SQL Not

    A condição lógica “not” é utilizada para acrescentar uma condição de negação, possibilitando a exclusão de registros na seleção;

    SQL And

    A condição lógica “and” é utilizada para acrescentar uma condição de inclusão, ou seja, o resultado será exibido apenas se todos os critérios de pesquisa forem obedecidos;

    SQL Or

    A condição lógica “or” é utilizada para acrescentar uma condição de alternativa, ou seja, o resultado será exibido caso seja obedecido o primeiro OU caso seja obedecido o segundo critério.
